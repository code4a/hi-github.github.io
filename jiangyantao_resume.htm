该应用是基于DLNA/UPNP协议开发的一款媒体共享软件，核心开发是基于自主开发的SDK提供相应的接口，拓展出相应平台的应用，基于UPNP协议通过C++编码，实现其五个部分Discovery / Description /Control / Eventing / Presentation ，通过(动态注册)JNI提供出主要的几个接口,SDK依赖这些接口封装出ControlPoint 从而控制设备分享媒体。基于SDK在WiMo客户端中的编码，在application中模拟任务栈，对其进行管理，提供一些耗时操作的方法，一些共有的方法，crash日志的收集等，抽取基类BaseActivity重写其每个生命周期，在onCreate中压栈，onDestory中弹栈，提供常用的变量，方法等简化代码，向下衍生出几个依赖wifi变化的activity基类，在其中统一管理相同的操作，不同的操作进行抽象，主界面添加原生的侧滑菜单Drawerlayout ，对菜单中的listview进行分栏展示。加入手势设别，添加手势库，监听用户手势，完成手势指令。媒体主界面采用FragmentActivity + fragment 完成tab标签的切换，相应的媒体列表采用AsyncTask异步读取SQLite数据库获取媒体信息，图片列表与媒体缩略图通过lrucache封装两级缓存管理内存避免列表卡顿到时anr和加载过多图片时oom。应用自动升级，检测局域网连通并且可访问外网的情况下进行升级，常规的http请求返回的结果相应较慢，当网络不可用时等待结果返回需要至少40s左右，而这40s内用户可能进行多次操作，而影响某些界面的显示，最终通过多种方法测试研究，可在3s内得到准确的结果
